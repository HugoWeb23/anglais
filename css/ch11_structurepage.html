Structure d'une page

Les balises structurantes d'HTML 5

L'HTML5 est arrivé avec toute une panoplie de nouveautés. Nous allons voir ici principalement les balises qui vont nous être utiles pour la structure de notre site. C'est à dire la balise à utiliser si l'on souhaite ajouter une en-tête, laquelle serait le plus adaptée pour le menu, etc.

Jusqu'à présent, pour diviser une page Web en plusieures parties, il fallait utiliser des <div> auxquelles on donnait un id précis. Ensuite, dans le fichier CSS, il suffisait d'appliquer une règle à cette <div> via son id.

Il a été constaté que souvent, pour les entêtes, on utilisait la balise <div> avec pour id header. La balise <header> a donc été créée.
Ce sera la même chose pour pieds de page avec <footer>, mais aussi d'autres balises encore donc voici la liste complète :

<header></header> : à utiliser en en-tête. Il peut y avoir plusieurs en-tête dans la page.
<footer></footer> : pour le pied de page. Idem que pour l'en-tête, on peut le mettre plusieurs fois dans la page.
<nav></nav> : pour les liens de navigation, destiné à contenir des menus de type <ul> par exemple.
<section></section> : pour regrouper les contenus par thèmes. Peut contenir des <header> ou <footer> par exemple.
<aside></aside> : infos complétentaires.
<article></article> : articles de journeux, de blog,...

Chaque section, ou en-tête peut contenir son titre <h1>.

Exemple

Exemple de code

<!doctype html>
<html lang="fr">

  <head>
    <meta charset="utf-8" />
    <title>Titre</title>
  </head>

  <body>

    <header>
      <h1></h1>
    </header>

    <nav>
      <ul>
        <li></li>
        <li></li>
      </ul>
    </nav>

    <section>
      <aside>
        <h1></h1>
        <p></p>
      </aside>
      <article>
        <h1></h1>
        <p></p>
      </article>
    </section>

    <footer>
      <p><a href="#">texte</a></p>
    </footer>

  </body>
</html>

Compatibilité IE

Le soucis avec IE, est qu'il ne reconnaît ces nouvelles balises comme étant des blocks. Il y aura donc des soucis à l'affichage, voire pas d'affichage du tout.
Pour remédier à ça, il suffit très simplement d'ajouter une ligne de code entre les balises <head> :

<head>
  <!--[if lt IE 9]>
    <script src="http://html5shiv.googlecode.com/svn/trunk/html5.js"></script>
  <![endif]-->
</head>

Cette ligne s'activera uniquement si la page est ouverte dans une version antérieure à IE9. Il lie la page vers un script en Javascript qui va s'occuper de convertir les balises pour qu'elles soient reconnnues.

Le positionnement

Float

Nous avons déjà eu un aperçu de ce que la propriété float pouvait apporter à nos images.
Nous allons à présent voir l'utilisation que nous pouvons en faire pour la structure de notre page Web.

Prenons comme exemple le cas d'une zone de navigation, qui contiendra les différents liens des pages de notre site, que l'on souhaite placer à gauche. Le contenu, lui, s'affichant sur la droite.

<!-- Tout d'abord le code HTML -->
<header>Mon entête</header>
<nav>
  Navigation
  <ul>
    <li>Lien 1</li>
    <li>Lien 2</li>
    <li>Lien 3</li>
  </ul>
</nav>
<section>
  Contenu de mon site ici.
</section>
<footer>Mon pied de page</footer>


/* A présent le css */
header, footer, nav, section {
  border: 1px solid #000;
}

nav {
  width: 150px;
  float: left;
}

section {
  height: 150px;
  width: 400px;
}
Mon entête
Navigation
  • Lien 1
  • Lien 2
  • Lien 3
Contenu de mon site ici.
Mon pied de page

Ca ne ressemble pas encore à ce que l'on voudrait obtenir. Il faudrait trouver une solution pour éloigner le block de contenu, de la zone de navigation.
Pour cela, deux solutions sont envisageables.

La première serait de donner au bloc <section> une marge gauche un peu plus grande que la largeur du bloc <nav>.
Dans ce cas-ci était donné que nav possède une width de 150px, il faudrait donc donner une margin-left de plus de 150px à notre section.

La seconde est de faire flotter le contenu à gauche également. Il viendra de la sorte se plaquer contre la zone de navigation sans passer dessous.
Mais il ne faudra pas oublier, dans ce cas là, de dire à notre pied de page qu'il doit ignorer le flottement des deux blocs précédents, et ce en lui appliquant un clear: both ou left.

Display

La propriété display est une propriété qui va nous permettre de contraindre nos éléments d'un type vers un autre.
Par exemple, transformer une balise <li> qui est de type block en un type de balise inline : nos <li> se positionneront sur la même ligne. Pratique pour faire un menu horizontal par exemple.

display peut prendre différentes valeurs :
block : les uns endessous des autres, avec espacement, comme pour les balises <p>.
inline : à l'inverse, transforme les balises block en balises de type inline, elles s'afficheront donc les unes à côté des autres.
inline-block : un mix des deux. Les éléments s'afficheront sur la même ligne, mais auront les propriétés d'une balise de type block !
none : va masquer la balise.

Inline-block

Voyons voir un peu plus en détails comment fonctionne la valeur inline-block.

<p>Premier bloc paragraphe.</p>
<p>Deuxième bloc paragraphe.<br /> Et un retour à la ligne.</p>

//---

p {
  display: inline-block;
  border: 1px solid #000;
}

Dans ce cas-ci donc, je suis sensée me retrouver avec deux paragraphes, l'un à côté de l'autre.

Deuxième bloc paragraphe.
Et un retour à la ligne.

Super ! Ca marche ! Mes deux paragraphes sont bien l'un à côté de l'autre !
Maintenant, je constate que l'alignement vertical par défaut se trouve vers le bas. Mais heureusement, il est possible de modifier ça grâce à la propriété vertical-align.

baseline : valeur par défaut, les blocs sont alignés avec la base de l'élément qui les contient (=le parent).
top : aligne en haut.
middle : aligne au milieu.
bottom : aligne en bas.
10%: valeur en px ou %, aligne à telle valeur par rapport à la ligne de base.

Cette valeur est implémenté dans IE à partir de la version 8. Pour les versions précédentes, il va falloir créer une feuille de style et mettre une condition dans l'entête du document html.
Pour en savoir plus, voir sur le site d'Alsacréations.

Voici à quoi ressemblerais la condition en question :

<!--[if lte IE 7]>
<link rel="stylesheet" href="style_ie.css" />
<![endif]-->

Dans ce cas-ci, je lui dis : "si le navigateur utilisé est plus ancien que IE7, alors utilise la feuille de style style_ie.css".
La feuille de style en question contiendra des adaptations qui auront été mises en place, spécialement pour ces navigateurs.

Positions absolue, relative et fixe

Nous allons à présent passer dans ce qui est, à mon sens, la propriété la plus intéressante des CSS. Je vous présente donc, la merveilleuse, la formidable, propriété position.

Tout ce qui est merveilleux demande un peu de réflexion et beaucoup de pratique, pour parvenir à le comprendre. Mais une fois que le principe en est acquis, toutes les possibilités s'offrent à nous.

Il y a trois sortes de positionnement :
absolute : le positionnement absolu, nous permet de placer notre élément n'importe où sur la page. Les éléments qui se trouvent derrière vont en profiter pour "prendre sa place".
fixed : comme pour la position absolue, la fixe nous permet également de placer l'élément n'importe où sur la page, sauf que cet élément restera fixe (voire la propriété background-attachment: fixed).
relative : enfin, il nous sera possible de décaler un élément par rapport à sa position initiale. Les éléments suivant, resteront bien gentillement à leur place.

Rien de tel que de tester cette propriété.

div {
  position: absolute;
  top: 10px;
  right: 150px;
}

Dans cet exemple, mon block ira se place à 10px du bord haut et à 150px du bord droit. En faisant défiler le texte, le block va défiler avec lui.

En testant les mêmes valeurs avec les autres possibilités de positionnement, vous verrez des résultats différents.
Si vous souhaitez positionner des éléments par rapport à leur parent, il faudra positionner cet élément parent (en lui donnant une position relative, par exemple, sans lui préciser de déplacement).

Si plusieurs de vos blocks doivent se superposer et que vous désirez contrôler lequel apparaît audessus des autres, il suffit pour cela d'utiliser la propriété z-index simplement en lui donnant une valeur.
Le block avec le z-index le moins élevé se retrouvera endessous et le plus élevée audessus.

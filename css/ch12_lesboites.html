Les boîtes

Block et Inline

Quelle est donc la différence entre les balises block et les balises inline desquelles on entend parler depuis le début ?

Balise de type block : elles vont créer un espace avant et après l'élément. Les balises vont donc se superposer, comme pour les <p>.
Balise de type inline : elles se trouvent toujours dans une balise de type block. Elle s'écrit à la suite du texte précédent et donc sur la même ligne. Comme pour la balise <a> par exemple.

Les balises de type inline sont par exemple : <a> <em> <strong> <mark> <img />
Les balises de type block sont par exemple : <p> <h1> <li> <section> <article>

<span> et <div> sont toutes les deux des balises universelles. La première est une balise de type inline et la seconde de type block.

Il peut être tentant de vouloir utiliser <span> et <div> partout, mais il est déconseillé de le faire.
Puisque des balises sont prévues pour ça, il vaut mieux utiliser la balise <strong> au lieu de <span id="important">.
Ou encore, se servir de la balise <h1> au lieu d'un <div id="titre">.
Le résultat sera probablement le même, mais ne serait-ce que pour un bon référencement, utilisez les balises pour ce dont elles sont prévues.

Dimensions

Il est possible de donner une taille aux blocks. Pour ce faire, nous allons utiliser les propriétés width et height.

p {
  width: 50%;
  height: 200px;
}

Dans l'exemple ci-dessus, on constate qu'il est possible de définir la taille soit en pourcentage, soit en pixels.
Par défaut, la hauteur height est définie à auto et la largeur width à 100%.

Gardez bien à l'esprit qu'il s'agit ici de la taille du contenu du block !
C'est à dire que si par la suite vous décidez d'ajouter une bordure et/ou une marge, par exemple, la taille de celles-ci vont venir s'ajouter à la taille du contenu du block.

Min et Max

p {
  width: 50%;
  max-width: 400px;
}

Les propriétés min-width, max-width, min-height et max-height, vont nous permettre de rendre notre site plus fluide, tout en gardant le contrôle sur son affichage.

Marges

Un texte complètement collé contre le bord d'une fenêtre de navigation, ce n'est pas très joli. Fort heureusement, en HTML, il est possible d'ajouter des marges à nos éléments.

Deux types de marges : les marges intérieures et les marges extérieures.

Marges intérieures

Ci-dessus, un paragraphe avec une marge intérieure de 10px, et un autre sans.
Ces marges intérieures sont définies à l'aide de la propriété padding.

p.partout {
  padding : 10px;
  /* les marges sont identiques des 4 côtés : haut, bas, gauche et droite */
}

p.vh {
  padding : 10px 20px;
  /* la marge en haut et en bas est de 10px, les gauche et droite sont de 20px */
}

p.differentes {
  padding : 10px 0px 30px 20px;
  /* dans l'ordre : haut, droite, bas, gauche. Comme pour les bordures */
}

p.separees {
  padding-right : 10px;
  padding-left : 20px;
  padding-top : 30px;
  padding-bottom : 40px;
  /* il est possible d'agir sur chaque côté séparément */
}

Les marges extérieures fonctionnent de la même manière mais en utilisant cette fois-ci la propriete margin.

Marges extérieures

Dans cet exemple, mon paragraphe entouré d'une bordure rose, ne possède pas de marges intérieures, mais bien une marge extérieure de 30px qui le sépare du block qui le contient (bordure noire).
Il est possible de centrer un paragraphe dans un bloc en lui définissant une largeur et en lui attribuant une marge extérieur avec la valeur auto.

De haut en bas :
visible : valeur par défaut, le texte qui dépasse du paragraphe reste visible.
hidden : le texte dépasse encore, mais il est coupé, impossible de lire la suite.
scroll : que le texte soit trop long ou pas, nous forçons l'affichage d'une scrollbar. Ici, il y en a une dans le bas qui est ajoutée alors que nous n'en avions pas besoin.
auto : visibilité automatique. Nous laissons le choix au navigateur d'afficher ou pas une scrollbar. Le texte ne dépasse plus, mais grâce à l'ascenceur il est possible de lire la suite.

Comme vous pouvez le constater, il vaut donc mieux mettre la valeur auto par défaut.

Il existe une balise <iframe> qui pourrait donner le même résultat, mais elle ne fonctionne pas de la même manière.
En effet, une <iframe> affiche le contenue d'une autre page HTML dans la page actuelle. A consommer avec modération.
